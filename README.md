# Uno Soft Java Test Task

Решение тестового задания на вакансию Java-разработчика: поиск непересекающихся групп строк по совпадению **непустых** значений в **одинаковых колонках**.

## Что делает программа

1. Читает входной файл построчно.
2. Валидирует формат строк вида `"..." ; "..." ; ...`.
3. Удаляет дубликаты (обрабатываются только уникальные валидные строки).
4. Строит группы по правилу:
   - если строки имеют одинаковое непустое значение в одной и той же колонке, они в одной группе;
   - учитывается транзитивность (A связан с B, B с C => A, B, C в одной группе).
5. Выводит только группы с размером > 1.
6. Сортирует группы по убыванию размера.

```bash
mvn clean package
```

## Запуск

Базовый запуск (как в условии):

```bash
java -Xmx1G -jar target/uno-soft-test-task-1.0.0.jar /absolute/path/to/lng-4.txt
```

С явным файлом вывода:

```bash
java -Xmx1G -jar target/uno-soft-test-task-1.0.0.jar /absolute/path/to/lng-4.txt /absolute/path/to/output.txt
```

Если второй аргумент не указан, результат пишется в `output.txt` в текущей директории.

## Формат входных данных

Ожидается формат:

```text
"value1";"value2";"value3"
```

- количество колонок в строке не ограничено;
- пустые значения (`""`) не участвуют в объединении;
- некорректные строки пропускаются.

Примеры некорректных строк:

```text
"8383"200000741652251"
"79855053897"83100000580443402";"200000133000191"
```

```text
Группа 1
строка1

...

Группа 2

```

## Алгоритм

Используется DSU (Union-Find):

1. Каждой уникальной валидной строке назначается `id`.
2. Для каждой колонки хранится `Map<value, firstRowId>`.
3. При повторной встрече `value` в той же колонке выполняется `union(currentId, firstRowId)`.
4. В конце все строки собираются по `root` компонент.
5. В вывод идут только компоненты с размером > 1.

Оценка сложности:
- время: `O(N * K * α(N))`, где:
  - `N` — число уникальных валидных строк,
  - `K` — среднее число колонок в строке;
- память: линейная относительно числа уникальных строк и количества уникальных значений по колонкам.

## Тестирование

Запуск unit-тестов:

```bash
mvn test
```

Покрыты основные сценарии:
- парсинг валидной и невалидной строки;
- транзитивное объединение строк в группу;
- отсутствие объединения при совпадении значений в разных колонках;
- игнорирование дубликатов и некорректных строк;
- сортировка групп по убыванию размера.
